เริ่มเขียนโปรแกรมด้วยตัวเองเถอะ

ผมเรียนจบแล้วเริ่มทำงานได้เกือบครบปีแล้ว ผมเริ่มงานจากดูแลระบบก่อนผันตัวมาเขียนโปรแกรมในที่สุด แต่ด้วยมุมมองนี้ทำให้เห็นปัญหาหลายอย่างของโปรแกรมเมอร์

ค้นหา หยิบมา แล้ว…

คุณเริ่ม googling ด้วยคีย์เวิร์ดที่ต้องการ เลือกมาสักหนึ่งหรือสองกระทู้จาก stack-overflow ทำการ copy, paste โปรแกรมทำงานได้อย่างที่คาดหวังแล้วนิทานเรื่องนี้ก็ตัดจบลงดื้อๆ

การใช้สิ่งที่มีอยู่แล้วไม่ว่าจะเป็น dependency หรือ code on the shelf ช่วยลดระยะเวลาและทำให้โปรแกรมเสร็จเร็วขึ้นก็จริง แต่คุณเองได้อะไรไหม

ไม่เริ่มเอง

คุณได้รับมอบหมายให้เขียนโปรแกรมด้วย node และไม่เคยเขียนมันมาก่อน คุณเริ่ม googling ด้วยคีย์เวิร์ด nodejs tutorial อ่านลิงก์แรกแบบคร่าวๆ แล้วปิด อ่านลิงก์ที่สองแบบเดียวกันแล้วปิด ทำอย่างนี้ไปเรื่อยๆ จนเจอโปรเจคตัวอย่างที่คล้ายกับสิ่งที่ต้องการแล้วลงมือดัดแปลงมันกับงานของคุณ

ผมไม่ได้เถียงว่าใช้ style guide หรือ skeleton project เป็นต้นแบบนั้นไม่ดี แต่จะดีกว่าไหมถ้าให้เวลาได้ getting started เพื่อทำความเข้าใจกับมันเสียก่อน ลองผิดลองถูกดูบ้างจะเป็นไรไป

ไม่ลึกจริง

ตอนนี้คุณเขียน node เป็นแล้ว ด้วยระบบ dependency module ทำให้คุณสามารถเลือกใช้สิ่งที่คนอื่นเขียนแล้วได้อย่างง่ายและประหยัดเวลา แต่ว่ามี feature หนึ่งที่ไม่สามารถหา module ได้ตรงความต้องการ คุณใช้เวลาเป็นวันๆ ในการ googling โดยหวังว่าจะเจอมัน

แล้วรุ่นพี่ senior ก็เดินเข้ามาพร้อมชี้สิ่งที่เรียกว่า api document ให้อ่าน มันเป็นคู่มือที่บอกถึงฟังก์ชันที่คุณเคยเรียกใช้มาแล้วเป็นสิบครั้ง แต่คุณไม่รู้ว่ามันทำแบบนี้ได้ด้วย ทุกอย่างจบลงด้วยการเติม parameter เข้าไปแค่ตัวเดียว โปรแกรมทำงานได้อย่างที่ต้องการ คุณได้บทเรียนว่าควรเริ่มเห็นประโยคของเอกสารที่คนอื่นเขียนไว้

ชอบเทสมือ

คุณได้เขียนโปรแกรมนี้มาเรื่อยๆ ด้วยการเขียน เซฟ รัน แล้วทดสอบด้วยการเปิดเว็บดู ตราบที่ยังไม่พอใจคุณก็วนทำขั้นตอนนี้แบบซ้ำๆ แล้วเพื่อนในทีมที่เขียนโปรแกรมมานานกว่าคุณก็ทักว่าเขียน automated test สิ คุณแอบ googling ดูเพื่อหาว่ามันคืออะไร แต่ก็ยังยืนยันว่ายุ่งยากแล้วทดสอบด้วยมือต่อไปจนเขียนฟังก์ชัน A เสร็จ

ต่อมาคุณได้รับมอบหมายว่าแยกออกเป็นฟังก์ชัน A แบบปรับปรุงและฟังก์ชันใหม่ B , C, D, … Z ทั้งหมดมีวิธีการทดสอบเหมือนกัน แตกต่างเพียงค่าที่ input ณ ตอนนี้เองคุณอยากจะได้ automated test ขึ้นมาแล้ว

เอกสาร เอกสารอะไร

คุณได้พัฒนาโปรแกรมจนเสร็จแล้วมอบงานให้ลูกค้า แต่เพียงไม่กี่ชั่วโมงจากนั้นคุณถูกโวยวายเนื่องจากเมื่อลูกค้าจะปรับแต่งมันเพียงนิดหน่อยแต่คุณไม่ได้เตรียมเอกสารอะไรไว้เลย คุณจึงต้องงมโปรแกรมของตัวเองพร้อมรับฟังเสียงบ่นของลูกค้าไปพร้อมๆ กัน ในขณะที่เพื่อนในทีมกำลังจิบกาแฟสบายใจเพราะลูกค้าแก้ไขเองได้โดยอ้างอิงคู่มือ

เรื่องราวทั้งหมดคือเรื่องที่เกิดขึ้นจริงกับผมเอง แต่ตัวละครอื่นสมมุตินะ ตอนนี้ผมปรับเปลี่ยนวิธีในการเรียนรู้ของตัวเองใหม่ ผมรู้สึกได้ว่าทำงานได้ง่ายขึ้น และงานก็มีคุณภาพมากขึ้น ทั้งหมดนี้มันเกิดขึ้นเพราะได้ลงมือทำด้วยความเข้าใจไงล่ะ
