ไม่ใช่ตัวเลือก แต่เป็นเรื่องมาตรฐาน

ผมเพิ่งจะกลับจากงาน iCode 2017 เป็นงานแข่งเขียนโปรแกรมระดับมัธยมศึกษาที่จัดโดยภาควิชาคอมพิวเตอร์ คณะวิศวกรรมศาสตร์ มหาวิทยาลัยเชียงใหม่ ในงานแข่งเขียนโปรแกรมจะมีเรื่อง System environment อยู่ มันควรจะเป็นมาตรฐานที่ทุกคนใช้ร่วมกัน ไม่ใช่ตัวเลือกสำหรับบางคน

เอนทรีนี้เป็นความเห็นส่วนตัวของผมที่เคยจัด ไม่ได้เป็นความเห็นจากผู้จัดงานครั้งนี้

เนื่องจากเป็นการแข่งขัน ดังนั้นต้องมีกฏ กติกา ข้อบังคับอยู่แล้ว ภายใต้การแข่งขันในรูปแบบของ ACM-ICPC จะมีเรื่องของ System environment ครับ จะว่าด้วย Version, library, editor, etc. เพื่อให้ผู้แข่งขันมีความเท่าเทียมกัน

ภาษาที่ใช้เขียนโปรแกรมก็มีพัฒนาอยู่เรื่อยๆ เช่นเดียวกับระบบปฏิบัติการหรือโปรแกรมอื่นๆ ทำให้มีเรื่องของ Version เข้ามาเกี่ยว หลายภาษามีการพัฒนาไปจากตัวเก่าๆ มาก ตัวอย่างเช่น การเปรียบเทียบ Node.js 0.12 กับ 8.x ซึ่งจุดนี้เองที่ทำให้เกิดปัญหาของวันนี้

เรื่องมีอยู่ว่า ก่อนการแข่งขันมีการประกาศกฏต่างๆ ที่เกี่ยวข้องไว้ที่นี่ แต่มีประโยคหนึ่งเขียนว่า “กฎกติกาการแข่งขันอ้างอิงจากกฎการแข่งขันของ ACM-ICPC โดยคณะกรรมการจัดงาน ได้ปรับปรุงบางส่วนให้เข้ากับสภาพการแข่งขันในพื้นที่ปัจจุบัน” ถ้าอ่านจากข้อความดังกล่าวผมเข้าใจว่าระบบจะ Inherit มาจากกฏของ ACM-ICPC แล้วปรับปรุงบางอย่าง อันที่เพิ่มหรือเอาออกควรจะเขียนบอก นอกจากนั้นควรจะเหมือนของดั่งเดิม

เพียงแต่วันจริงไม่ได้เป็นอย่างที่เข้าใจครับ ส่วนการตั้งค่าที่เรียกว่า Compilation of Submissions ไม่ถูกนำมาด้วย ทำให้เกิดปัญหากับผู้แข่งขันที่ฝึกมากับกฏของ ACM-ICPC ในการ Compile ภาษา C++ ผลสุดท้ายผู้แข่งขันอาจจะรันที่เครื่องตัวเองได้ แต่เมื่อส่งผลไปจะได้ผลเป็น Complie error เพราะระบบจะ Complie ด้วย c++98 แทนซึ่งผมว่ามันไม่ยุติธรรมกับนะ ส่วนนี้น่าจะประกาศให้ชัดและบังคับใช้ทุกคน

ที่ผมเขียนเอนทรีไม่ใช่เพื่อจุดประสงค์ประท้วง ตำหนิ หรือร้องเรียน แต่อยากจดบันทึกไว้เพื่อไม่ให้เกิดขึ้นอีกในครั้งหน้า

ด้านล่างนี้เป็นส่วน Compilation of Submissions ตัวอย่างของ May 2017

C:
gcc -g -O2 -std=gnu11 -static $* -lm
C++:
g++ -g -O2 -std=gnu++14 -static $*
Java:
javac -encoding UTF-8 -sourcepath . -d . $*
Python 2:
python2 -m py_compile $*
Python 3:
python3 -m py_compile $*
