เรื่องราวของผมกับ javascript developer

tl;dr

เริ่มจากเขียน jquery, ajax ก่อนจะรู้จัก node แล้วเปลี่ยนใจเป็น backend เริ่มเรียนรู้ว่า ops ยังไงให้มันทำงานได้ทั้งบน self-managed server และ cloud platform ผมกลับมาเขียน front อีกครั้งด้วย ember, angularjs พร้อมพัฒนาการเขียนให้มีแนวทาง เริ่มใช้ task running, testing tool ได้หัดทำงานแบบ devops พร้อม docker และเริ่มเปิดโลกให้กว้างขึ้นกว่าการแค่ googling ในหน้าจอ

Client javascript

ผมว่าตัวนี้คือพื้นฐานสุดๆ สามารถเริ่มได้เลยจากการเปิด console บน web browser ผมเรียนรู้การเขียน function ง่ายๆ และต่อยอดหากินด้วย css selector บน jquery ที่ช่วยให้เพิ่ม action หรือ animate ที่ไม่สามารถทำได้บน php based

ลำพังแค่ความรู้ js คงจะไปไม่รอด ผมเริ่มอ่านให้ลึกขึ้นว่า html web ทำงานยังไง dom คืออะไร json หน้าตาเป็นอย่างไรทำไมถึงได้รับความนิยม

<button onClick=”myFunction()”>Click Me

Node

ต่อมาผมได้ลอง node แล้วหลงรักในที่สุดบนโปรเจคจบของผมเอง ตอนนั้นมี module อะไรให้ใช้ก็มาอย่างเดียว ไม่รู้เลยด้วยซ้ำไปว่าโปรแกรมเราเรียกมันได้อย่างไรผมได้ใช้ mongodb เป็นครั้งแรกร่วมกับ mongoose (library mongodb บน node) บน express (web server framework)

ผมเจอปัญหาว่าแค่ function ง่ายๆ ก็เขียนไม่ได้เพราะสิ่งที่เรียกว่า callback แต่ก็ยังประคอบแบบถูไถมาเรื่อยๆ จนเรียนจบมาได้พร้อมกับคำถามว่า promise ต่างกับ callback ยังไง

Callback from hell and pyramid of doom.

Running Node Web Server

ผมเรียนจบแล้ว เลยลองรับโปรเจค node backend มาหนึ่งชิ้นเป็น api แล้วทำงานคู่กับ frontend อีกทีม นั้นเป็นครั้งแรกที่ผมให้ทำงานแบบ front/back

สมัยเรียนผมรัน node ด้วย cli แต่ตอนนี้ต้องเป็น production แล้ว ด้วยการ googling อันเก่งกาจเลยได้ pm2, forever มาช่วยในการรันให้ โดยทำตัวเหมือน service ที่คอยดูแล node process อีกที พร้อมกับรู้จักการใช้ nginx มาเป็น reverse proxy ทำให้แม้จากข้างนอกเห็นว่าเป็น https แต่ก็จะสามารถใช้ node web server ที่รันอยู่หลังบ้านได้

Frontend Framework

ตอนนั้น (mid-2015) กระแส single page app รอบตัวผมมาแรงแซงโค้ง ผมเริ่มเขียนโปรแกรมบัญชีรายรับรายจ่ายบน ember js จนมาถึงโปรแกรมคลังหนังสือที่ซับซ้อนเพิ่มขึ้นบน angularjs ร่วมกับ express เจ้าเก่าในหน้าที่ backend

<div ng-show=”book.isSoldOut()”>…</div>

Style Guide Era

ไม่ว่าจะ frontend หรือ backend พอโปรแกรมเริ่มใหญ่ ความซับซ้อนมากขึ้น จำนวนไฟล์กลายเป็นสิบๆ ผมเลยศึกษาว่าชาวบ้านเค้าเขียนกันยังไงให้มันจัดการได้ สำหรับ node นั้นลองหาศึกษาได้จาก npm repo ได้ว่าเค้าเขียนยังไงให้เป็น modular แต่สำหรับ angularjs ผมอาศัยอ่านจาก style guide ของคนอื่นเอา

Task Runner

เป็นเหมือนสิ่งที่อยู่คู่กับ js เหมือนเล่นเกมคอมต้องมีการ์ดจอ ผมเริ่มจาก grunt แล้วก็เอ๋อไปอยู่ช่วงใหญ่ๆ ก่อนจะใช้ gulp มาจนถึงทุกวันนี้ แล้วตอนนี้กำลังเริ่มสนใจ webpack

Automated Test

mocha เป็นชื่อที่ผมพบในการ googling ในการทำ test บน express โดยมักถูกใช้คู่กับ assert library เช่น chai หรือ supertest ที่ใช้ในการทดสอบ api endpoint ถ้าให้โปรแกรมเมอร์มือใหม่แนะนำเด็กจบใหม่ที่อยากสบายบนสาย express ก็คู่นั้นแหละเหมาะที่สุดแล้ว

DevOps, Docker

พอตัวละครครบแล้ว โปรแกรมพร้อม เทสพร้อม ผมเริ่มศึกษาว่า continuous integration (ci), continuous deployment (cd), devops คืออะไร ผมเริ่มจาก heroku platform ที่มีการทำงานง่ายๆ ตาม Dockerfile; configurate file ที่บอกว่า container เราหน้าตายังไงและทำงานอะไร

พร้อมกันตอนนั้นผมได้มีโอกาสทำงานแบบ agile; scrum ตอนนั้นเหมือนตาสว่าง เหมือนเห็นอีกโลกหนึ่งที่ไม่เห็น สิ่งที่เคยเรียนในวิชา software engineering กลายเป็นเหมือนเครื่องจักรไอน้ำเมื่อมองจากยุคอวกาศ ใครไม่เป็นตอนนี้เชยแล้วนะ

$ git push heroku master

ออกสู่โลกกว้าง

ผมเริ่มร่วมงาน contribute ต่างๆ เริ่มไปงานพบปะอย่าง barcamp, beercamp แบบถี่ๆ ผมเริ่มเขียนและแบ่งปันคนอื่น ผมทดลองใช้ solution ต่างๆ ไม่ว่าจะเป็น aws, google cloud, azure หรือ platform as a service อื่นตลอด ผมเริ่มอ่านเรื่อง team software process และมีความอยากลองเป็นผู้นำมากขึ้น หวังว่าผมจะได้โอกาสนั้นในเร็วๆ นี้

แล้วทั้งหมดนี้ก็คือ javascript ในแบบของผม
